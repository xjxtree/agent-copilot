# V2.92 Verification Checklist

Status: completed

## Scope

V2.92 closes the Codex expanded roots slice without opening new write targets.

- [x] Codex expanded roots now expose verified user/project `.agents/skills`
  roots plus read-only `$CODEX_HOME/skills`, local plugin marketplace roots,
  and `/etc/codex/skills` when present.
- [x] `RootSource::Compatibility`, `RootSource::Admin`, and
  `RootSource::Plugin` are represented in adapter diagnostics and root labels.
- [x] Project `.codex/config.toml` appears in diagnostics, but toggle writes
  continue to use only the user `config.toml` override.
- [x] Local plugin marketplace roots are parsed only from local marketplace
  entries; escaping or remote plugin sources are skipped.
- [x] Codex writes remain allowlisted to user/project `.agents/skills`
  instances; `$CODEX_HOME/skills`, plugin marketplace, admin, and system roots
  are read-only.
- [x] No provider calls, provider default calls, script execution, credential read, cloud sync, telemetry, hidden write path, or network fetches were added.

## Source Evidence

- Official Codex Agent Skills docs identify repo/user/admin/system skill
  locations and user `[[skills.config]]` entries:
  <https://developers.openai.com/codex/skills>.
- Official Codex plugin docs describe local plugin source paths and plugin
  skill directories: <https://developers.openai.com/codex/plugins/build>.
- V2.92 intentionally treats plugin/admin/system/compatibility roots as
  scan/diagnostic evidence unless a later version proves rollback-safe writes.

## Safety Decision

Final status decision: completed.

V2.92 resolves Codex project config, compatibility roots, admin roots, and
plugin marketplace planning by implementing read-only discovery/diagnostics plus
a native-root write allowlist. It does not enable project `.codex/config.toml`
writes, plugin install hooks, network fetches, provider calls, script
execution, credentials access, cloud sync, telemetry, or hidden write/apply
paths.
