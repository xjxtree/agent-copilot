# V2.95 Verification Checklist

Status: completed

## Scope

V2.95 closes the Hermes native-root install slice without adding new service
methods or guessing Hermes config-toggle semantics.

- [x] Hermes native-root install supports confirmed local ToolGlobal
  `SKILL.md` copy into `~/.hermes/skills/<name>/SKILL.md`.
- [x] Adapter capability matrix reports Hermes as `install-only` with
  `verified-native-root-v2.95` install and `install-only-v2.95` writable
  status.
- [x] Hermes config toggles remain blocked because individual skill
  enable/disable schema and rollback-safe config writes are not verified.
- [x] Hermes `skills.external_dirs` remain read-only scan roots and are not
  writable or install targets; external_dirs remain read-only.
- [x] Hermes project-local install remains blocked because generic project
  skill discovery is not confirmed.
- [x] Hermes hub, URL, tap, update, uninstall, reset, and network-backed
  package operations remain blocked.
- [x] No provider calls, provider default calls, script execution, credential
  reads, cloud sync, telemetry, hidden write path, uncontrolled network fetch,
  or release artifact automation was added.

V2.95 uses the documented `~/.hermes/skills/` local skill root and manual
`SKILL.md` directory creation semantics. The docs describe per-platform
enable/disable through Hermes UI/CLI, but do not provide a stable config file
schema for rollback-safe individual skill toggles, so those writes remain
blocked.

## Safety Decision

Final status decision: completed.

V2.95 resolves Hermes native-root install by allowing only confirmed local
ToolGlobal `SKILL.md` copies into `~/.hermes/skills`. It does not enable Hermes
config toggles, per-platform enablement writes, `skills.external_dirs` writes,
project installs, hub / URL / tap / update / uninstall / reset operations,
provider calls, script execution, credential reads, raw prompt/response/trace
persistence, cloud sync, telemetry, uncontrolled network fetch, hidden
write/apply paths, signing, notarization, DMG, ZIP, or release automation.
