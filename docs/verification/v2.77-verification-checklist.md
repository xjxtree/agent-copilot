## Canonical Blocker Explanation Matrix

| Blocker | Workbench explanation requirement | Completion rule |
| --- | --- | --- |
| `locked-session` | Explains that the macOS session is locked or not interactive, so Computer Use and screenshot evidence cannot prove UI behavior. | Blocked until the session is unlocked and retried. |
| `window-not-found` | Explains that the target app window could not be resolved for the current bundle/PID. | Blocked until the current app window is targetable. |
| `no-ax-window` | Explains that the app process exists but no Accessibility window is available for interaction. | Blocked until AX exposes the app window. |
| `screen-recording-permission` | Explains that app-window capture is blocked by Screen Recording permission or related capture authorization. | Blocked until permission is fixed and capture is retried. |
| `stale-bundle` | Explains stale-bundle or duplicate bundle evidence, including same bundle id processes from another worktree or an older `dist/SkillsCopilot.app`. | Blocked until validation targets the current `<repo>/dist/SkillsCopilot.app`. |
| `black-capture` | Explains that the screenshot is black and cannot be accepted as UI evidence. | Blocked; invalid capture must not be committed as completion evidence. |
| `flat-capture` | Explains that the screenshot is near single-color/flat and cannot prove the UI. | Blocked; invalid capture must not be committed as completion evidence. |
| `transparent-capture` | Explains that the screenshot has transparent or effectively empty content. | Blocked; invalid capture must not be committed as completion evidence. |
| `invalid-capture` | Explains unreadable, corrupt, missing, non-PNG, or otherwise invalid screenshot evidence. | Blocked until a valid app-window screenshot is captured. |
| `computer-use-timeout` | Explains that Computer Use timed out before proving interaction with the app. | Blocked until unlocked Computer Use completes or a more specific canonical blocker is found. |
| `remote-connection` | Explains that Computer Use or AX failed through a remote-connection/tool transport issue. | Blocked until the tool layer can inspect the local app. |
| `activation-failed` | Explains that the current app could not be activated or brought forward for validation. | Blocked until activation and foreground targeting succeed. |
| `tool-layer-unknown` | Explains that an unclassified Computer Use, AX, or capture failure happened and needs diagnosis. | Blocked until classified more specifically or resolved. |
