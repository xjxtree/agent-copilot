# V2.97 Verification Checklist

Status: completed

## Scope

V2.97 closes the Agent Config center and the narrow Hermes/OpenClaw
configuration-toggle slice.

- [x] Agent Config shows each agent's capability, current config target, config
  snapshot history, preview, and rollback entry. The current app reaches this
  through the primary sidebar Config workflow rather than Settings.
- [x] Claude keeps its existing current-config editor.
- [x] Non-Claude raw config editing is not added; non-Claude writes stay on
  verified toggle paths.
- [x] Hermes toggles write only the documented global `skills.disabled` list in
  `~/.hermes/config.yaml`.
- [x] OpenClaw toggles write only documented global
  `skills.entries.<key>.enabled` values in `~/.openclaw/openclaw.json`.
- [x] OpenClaw config reads accept JSON5 input and write back strict pretty JSON.
- [x] Hermes/OpenClaw toggles use snapshot/read-back/rollback and rescan.
- [x] Scanner disabled-state readback honors Hermes `skills.disabled` and
  OpenClaw `skills.entries.<key>.enabled`; OpenClaw uses
  `metadata.openclaw.skillKey` when present.
- [x] Safe Batch can include Hermes/OpenClaw only through the same guarded
  preview-confirmed toggle path.
- [x] No provider default call, script execution, credential read/write, cloud
  sync, telemetry, hidden write path, uncontrolled network fetch, signing,
  packaging, DMG, ZIP, notarization, or release automation was added.

## Source Evidence

- Hermes FAQ documents `~/.hermes/config.yaml`, `skills.disabled`, and
  `platform_disabled`: <https://hermes-agent.nousresearch.com/docs/reference/faq>
- Hermes user guide documents native `~/.hermes/skills` roots:
  <https://hermes-agent.nousresearch.com/docs/user-guide/features/skills>
- OpenClaw skills configuration documents
  `~/.openclaw/openclaw.json`, `skills.entries.<key>.enabled`, and
  `metadata.openclaw.skillKey`: <https://docs.openclaw.ai/tools/skills-config>
- OpenClaw skills docs document native and workspace skills:
  <https://docs.openclaw.ai/tools/skills>

Local validation found no installed `hermes`, `openclaw`, or `claw` CLI in this
environment. The local `~/.openclaw/openclaw.json` is JSON5-style config
(`JSON.parse` rejects it), so V2.97 explicitly parses JSON5 before applying a
strict JSON write-back.

## Safety Decision

Final status decision: completed.

V2.97 supports only the researched, documented, and tested Hermes/OpenClaw skill
enable/disable configuration fields. It does not enable Hermes
`platform_disabled`, `external_dirs` writes, hub/URL/tap/update/uninstall/reset,
OpenClaw allowlists, env/apiKey, install policy, load roots, ClawHub, Git,
update, verify, workshop, network-backed operations, scripts, credentials,
cloud sync, telemetry, raw multi-agent config editing, or release automation.
