# V2.90 Verification Checklist

Status: completed on 2026-06-17.

V2.90 Internal identifier migration is completed as a compatibility-first slice.
It migrates the packaged macOS app identity and default app-data id to Agent
Copilot while preserving legacy automation, module, sidecar, AX, env-var, and
credential compatibility.

Closeout keyword: app-data compatibility migration.

## Scope

- New primary app bundle: `dist/AgentCopilot.app`.
- New `CFBundleName` / executable: `AgentCopilot`.
- New primary bundle id and service default app-data id:
  `dev.agent-copilot.native`.
- Legacy bundle id retained for compatibility and migration:
  `dev.skills-copilot.native`.
- Legacy app-data directories are copied to the new default directory only when
  the new directory is absent.
- Migration marker:
  `agent-copilot-app-data-migration.json`.
- Explicit `SKILLS_COPILOT_APP_DATA_DIR` bypasses default migration.
- `script/build_and_run.sh`, `script/capture_app_window.sh`, and
  `scripts/smoke-macos-app.mjs` now target the Agent Copilot bundle by default
  and still detect legacy `SkillsCopilot` processes for cleanup/ambiguity.

## Compatibility Boundaries

- Swift package, product, target, source directory, and test module names remain
  `SkillsCopilot`.
- Rust crate and package names remain `skills-copilot-*`.
- Packaged sidecar remains `skills-copilot-service`.
- AX identifiers remain `skills-copilot.*`.
- Environment variables remain `SKILLS_COPILOT_*`.
- Keychain service remains `dev.skills-copilot.native.llm` to avoid silent
  credential copying.
- Historical evidence and completed version checklists keep their original
  `SkillsCopilot` paths and AX identifiers.

## Safety Boundary

The V2.90 migration is local-only. It does not delete legacy app data, does not
copy Keychain secrets, does not rename AX ids or env vars, and does not add
provider calls, network calls, product script execution, skill-file writes,
agent-config writes, snapshot/rollback mutation, raw prompt/response/trace
persistence, cloud sync, telemetry, or release packaging.
