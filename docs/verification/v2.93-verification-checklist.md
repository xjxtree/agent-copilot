# V2.93 Verification Checklist

Status: completed

## Scope

V2.93 closes the opencode custom roots slice by implementing custom local roots
without opening URL fetches or new write/install targets.

- [x] opencode now scans native roots, official `.claude` / `.agents`
  compatibility roots, and configured local `skills.paths` roots.
- [x] Configured roots use `RootSource::Configured` and are read-only scan
  sources.
- [x] JSON and JSONC opencode config files are read for `skills.paths`, with
  expansion, canonicalization/dedupe, and project-boundary checks.
- [x] `skills.urls` is metadata-only; scans and diagnostics do not fetch
  network skill indexes.
- [x] macOS provenance labels include configured local paths.
- [x] opencode writes remain exact managed `permission.skill` overrides, and
  installs remain native-root-only.
- [x] No provider calls, provider default calls, script execution, credential
  reads, cloud sync, telemetry, hidden write path, URL fetch, or configured-root
  write/install target was added.

## Safety Decision

Final status decision: completed.

V2.93 resolves opencode custom local roots by implementing read-only
configured local paths, configured provenance, and no-fetch URL handling. It
keeps a no uncontrolled network boundary and does not enable `skills.urls`
network discovery, configured-root writes,
configured-root installs, provider calls, script execution, credential reads,
cloud sync, telemetry, hidden write/apply paths, signing, notarization, DMG, or
ZIP work.
