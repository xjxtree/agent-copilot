# V2.96 Verification Checklist

Status: completed

## Scope

V2.96 closes the OpenClaw native/workspace install slice without adding new
service methods or guessing OpenClaw config-toggle semantics.

- [x] OpenClaw native-root install supports confirmed local ToolGlobal
  `SKILL.md` copy into `~/.openclaw/skills/<name>/SKILL.md`.
- [x] OpenClaw workspace install supports confirmed local ToolGlobal
  `SKILL.md` copy into `<workspace>/skills/<name>/SKILL.md` only when the
  selected project is the confirmed OpenClaw workspace or inside it.
- [x] Adapter capability matrix reports OpenClaw as `install-only` with
  `verified-native-workspace-v2.96` install and `install-only-v2.96` writable
  status.
- [x] OpenClaw `.agents` roots remain scan-only and are not direct install
  targets; .agents roots remain scan-only.
- [x] OpenClaw config toggles and `skills.entries` writes remain blocked
  because credential-safe JSON/JSON5 patching and rollback semantics are not
  verified.
- [x] OpenClaw ClawHub, Git, update, verify, workshop, and network-backed
  operations remain blocked.
- [x] No provider calls, provider default calls, script execution, credential
  reads, cloud sync, telemetry, hidden write path, uncontrolled network fetch,
  or release artifact automation was added.

## Safety Decision

Final status decision: completed.

V2.96 resolves OpenClaw native/workspace install by allowing only confirmed
local ToolGlobal `SKILL.md` copies into `~/.openclaw/skills` or confirmed
OpenClaw workspace `<workspace>/skills`. It does not enable OpenClaw config
toggles, `skills.entries` writes, `.agents` direct installs, ClawHub, Git,
update, verify, workshop, provider calls, script execution, credential reads,
raw prompt/response/trace persistence, cloud sync, telemetry, uncontrolled
network fetch, hidden write/apply paths, signing, notarization, DMG, ZIP, or
release automation.
