# V2.94 Verification Checklist

Status: completed

## Scope

V2.94 closes the Pi install and compatibility-root writes slice without adding
new service methods.

- [x] Pi now scans native `~/.pi/agent/skills` / project `.pi/skills` roots
  plus `.agents/skills` compatibility roots.
- [x] Pi `.agents/skills` roots use `RootSource::Compatibility`.
- [x] Pi compatibility-root toggles update Pi settings, not `.agents` skill
  files.
- [x] Project compatibility toggles use project-bound settings snapshots and
  block explicit untrusted markers; 2026-06-19 real local validation confirmed
  project `.pi/skills` loads without a positive `project.trusted` marker.
- [x] Tool-global install supports native-root install to
  `~/.pi/agent/skills` and project `.pi/skills`.
- [x] Pi package install/remove remains blocked.
- [x] `.agents/skills` direct skill-file installs remain blocked.
- [x] No provider calls, provider default calls, script execution, credential
  reads, cloud sync, telemetry, hidden write path, package install/remove, or
  uncontrolled network fetch was added.

## Safety Decision

Final status decision: completed.

V2.94 resolves Pi install and compatibility-root writes by allowing only
guarded Pi settings toggles for native / `.agents` compatibility instances and
native-root direct installs. It does not enable package install/remove,
`.agents` direct skill-file installs, provider calls, script execution,
credential reads, raw prompt/response/trace persistence, cloud sync,
telemetry, uncontrolled network fetch, hidden write/apply paths, signing,
notarization, DMG, or ZIP work.
