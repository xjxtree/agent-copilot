# V2.91 Verification Checklist

Status: completed on 2026-06-17.

V2.91 Model-task matching history is completed as a local evidence-domain
slice. It records redacted model/task fit metadata in app data, combines that
metadata with existing redacted prompt-run records, and surfaces the resulting
history read-only in Provider Observability.

Closeout keyword: Model-task matching history.

## Scope

- New app-local metadata file: `model-task-matches.json`.
- New service methods:
  - `llm.listModelTaskMatches`
  - `llm.recordModelTaskMatch`
  - `llm.deleteModelTaskMatch`
- `llm.listModelTaskMatches` is read-only and combines app-local match records
  with redacted prompt-run metadata.
- `llm.recordModelTaskMatch` writes only redacted app-local metadata and rejects
  empty task/model records.
- `llm.deleteModelTaskMatch` deletes only one app-local metadata record by id.
- `llm.providerObservability` now includes `model_task_history_rows`.
- Provider Observability shows model-task history read-only; V2.91 adds no
  native write/delete controls.

## Non-goals

- No provider request, provider default call, or network call.
- No credential read or Keychain migration.
- No raw prompt, raw response, raw trace, secret, or unredacted local path
  persistence.
- No write path beyond the explicitly scoped app-local redacted metadata
  record/delete methods.
- No skill file write, agent config write, snapshot mutation, triage mutation,
  or hidden write/apply path.
- No script execution path exposed to the product.
- No cloud sync.
- No telemetry.
- No signing, notarization, DMG, ZIP, updater, or public distribution work.

## Safety Boundary

The V2.91 history domain is local-only. Record/delete methods mutate only
redacted app-local `model-task-matches.json` metadata, and the V2.91 native UI
is read-only. The slice does not add provider calls, credential reads, raw
prompt/response/trace persistence, skill/config writes, snapshots, triage
mutation, script execution, cloud sync, telemetry, or distribution packaging.
