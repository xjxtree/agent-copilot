# V2.81 Verification Checklist

Status: completed on 2026-06-15.

V2.81 Swift stdio sidecar cancellation cleanup is completed. It hardens cancellation and timeout cleanup around the existing short-lived JSON stdio service sidecar without changing the service protocol, adding a daemon/socket layer, or changing user-visible native UI surfaces.

## Scope

- Swift service IPC cleanup: harden `ServiceClient.runService` cancellation and timeout behavior around the current short-lived stdio sidecar.
- Process cleanup: cancelled or timed-out service calls should terminate the child process and avoid leaked pipe handles.
- Pipe handling: stdin/stdout/stderr handles are closed during cleanup when the task is cancelled or the service exits early.
- Protocol stability: keep the current JSON service protocol method names and payloads.
- Gate parity: `pnpm verify:v2.81-docs` is a completed verifier and is part of the shared `pnpm verify:gate-parity` lane.

## Non-goals

- No daemon or socket redesign by default.
- No service protocol method additions, renames, removals, or payload changes.
- No provider default calls or background network calls.
- No write/apply path.
- No hidden task state.
- No scanner/catalog fact mutation.
- No script execution.
- No credential reads.
- No raw prompt, raw response, or raw trace persistence.
- No cloud sync.
- No telemetry.
- No public distribution.
- No signing, notarization, DMG, or ZIP work.
