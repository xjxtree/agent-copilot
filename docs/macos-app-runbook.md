## When `dist/AgentCopilot.app` Is Updated

`dist/AgentCopilot.app` is regenerated whenever `script/build_and_run.sh` runs in `run` or `--verify` mode. The script:

- builds `skills-copilot-service`
- builds the SwiftPM macOS app
- removes and recreates `dist/AgentCopilot.app`
- copies the Swift app binary into `Contents/MacOS/AgentCopilot`
- copies the Rust service sidecar into `Contents/Resources/skills-copilot-service`
- copies Swift localized resources from `apps/macos/Sources/SkillsCopilot/Resources` into `Contents/Resources`
- copies `apps/macos/Sources/SkillsCopilot/Resources/AppIcon.icns` into `Contents/Resources/AppIcon.icns`
- writes `Contents/Info.plist`
- launches the app

Therefore these commands update `dist/AgentCopilot.app` before launching or checking it:

```sh
./script/build_and_run.sh run
./script/build_and_run.sh --verify
pnpm dev
pnpm build
pnpm dev:macos
pnpm build:macos
pnpm check:macos
```

`pnpm smoke:macos-app` does **not** build or package the app. It only validates the existing `dist/AgentCopilot.app`.

By default, Smoke App Run checks freshness before it launches:

- Swift sources / `apps/macos/Package.swift` must not be newer than `Contents/MacOS/AgentCopilot`
- Rust workspace sources / Cargo manifests must not be newer than `Contents/Resources/skills-copilot-service`
- Swift localized resources must not be newer than `Contents/Resources/en.lproj/Localizable.strings`
- `apps/macos/Sources/SkillsCopilot/Resources/AppIcon.icns` must not be newer than `Contents/Resources/AppIcon.icns`
- `script/build_and_run.sh` must not be newer than the generated app binary, sidecar, icon, or `Info.plist`

If any input is newer, Smoke App Run fails and tells the developer to run `./script/build_and_run.sh --verify` or `pnpm check:macos`. A diagnostic-only escape hatch exists: `pnpm smoke:macos-app -- --allow-stale-app`, but normal validation must not use it.

- opencode fixture roots are isolated to temporary roots: native `~/.config/opencode/skills` / project `.opencode/skills` plus official `.claude` / `.agents` compatibility roots when covered by the scenario; smoke must not create or scan real user opencode config
- when the integrated service reports opencode rows from `catalog.scanAll`, smoke verifies global opencode skills without project context, project opencode skills with project context, and `config.toggleSkill` rejection/read-only behavior
- V2.9 tool-global validation is covered by Rust service tests, Swift UI/model tests, layout checks, and service fixtures: tool-global rows must display as read-only previews, toggle must remain disabled, import writes only app-controlled staging/catalog, export manifests remain reproducible, and install/copy to Claude/Codex must be confirmation-gated through verified write paths
- V2.10 execution safety validation is a boundary check, not a runner check.
- Smoke, scan, import, export, install, LLM prepare, state snapshot, and detail loading must not execute skill scripts.
- Any execution affordance must remain default-denied and require a future per-request preview/confirmation/audit path.

Smoke App Run must not use real user config.

Fixture mode uses only temporary `HOME`, app data, and project roots. It must avoid real Claude, Codex, and opencode config.

Screenshot privacy rules:

- Smoke and real-local screenshots must not expose actual local `HOME`, app data, `/var/folders`, temporary fixture roots, tokens, or credential placeholders.
- Keep screenshot privacy mode enabled by default for committed evidence. Native UI path rows should show screenshot-safe placeholders and collapsed paths unless the reviewer explicitly reveals a full path for local debugging.
- `script/capture_app_window.sh` fails fast for locked macOS sessions, selects the exact app window by CG/AX metadata, captures it with `/usr/sbin/screencapture -l <window_id> -o -x <out.png>`, and rejects near-black, mostly transparent, or near-flat captures with canonical blocker codes.
- Use `pnpm classify:validation-blocker -- "<tool output>"` when Computer Use, AX, or direct capture returns raw text such as `timeoutReached`, `cgWindowNotFound`, or `remoteConnection`.
- Run `pnpm verify:screenshot-artifacts` after regenerating screenshots.
- Redact visible sensitive values before committing evidence.
- Run `pnpm check:privacy`.
- New screenshots still require manual visual inspection.

- protocol v1 is default-deny for execution: `script.previewExecution` and `script.execute` are supported for safe preflight and confirmation-gated attempt flow, while real script execution is deferred until a sandbox runner is shipped.
- scan, import, export, install, LLM prepare, state snapshot, detail loading, and smoke fixture flows do not spawn skill scripts or helper commands from skill content
- any future execution affordance requires per-request user confirmation and shows cwd/env/network/files preview before execution can be considered
- blocked, cancelled, and failed attempts have an audit record contract; `completed` execution records remain invalid until a real sandbox runner exists
- LLM output cannot trigger execution, set confirmation, or bridge into execution-like service calls

The current mainline app passed real local Computer Use on 2026-06-09. The pass verified that the visible UI does not present script execution as a completed working feature; the real catalog had no structured script command records, so the preview reached a safe missing-command preview-only state without execution. If an execution affordance changes in a future build, verify it is default-denied, preview-first, confirmation-gated, and records blocked/cancelled/failure attempts without running the script unless the real sandbox runner has separately shipped. Fixture smoke screenshots and launch verification do not close this real local validation item for future candidates.

## Screenshot Rule

Completed UI screenshots must be app-window-only captures. Use:

```sh
pnpm capture:macos-window
```

or:

```sh
script/capture_app_window.sh AgentCopilot docs/ui-artifacts/native-macos-shell/completed.png
```

Full desktop screenshots are forbidden.
Visible local paths, usernames, app data directories, temp directories, tokens, and credential placeholders are also forbidden in committed screenshots. Use redacted screenshots for repository artifacts and keep raw local captures out of git.

## Current Non-Goals

- No public distribution artifact
- No signed app requirement
- No notarization requirement
- No DMG / ZIP packaging requirement
- No public distribution naming
- No completed LLM provider client, network-call, or credential-storage requirement in the V2.7 gate boundary
- No completed script runner, sandbox runtime, successful execution output log, or LLM-triggered execution path in the V2.10 safety boundary

See [`distribution-runbook.md`](./distribution-runbook.md) for the V2 Prep checklist that must be completed before these become release requirements.
