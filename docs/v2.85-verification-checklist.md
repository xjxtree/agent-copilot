# V2.85 Verification Checklist

Status: completed on 2026-06-16.

V2.85 Rust RPC domain module splitting is completed. This is a refactor-only service module-boundary slice that keeps the existing 88-method service protocol, request/response payloads, dispatch behavior, and provider/write/script/cloud/telemetry boundaries unchanged.

## Scope

- `ServiceHost` RPC implementation moved out of `crates/service/src/lib.rs` into domain files: `service_host.rs`, `service_cleanup.rs`, `service_knowledge.rs`, `service_llm.rs`, `service_remediation.rs`, and `service_task.rs`.
- `crates/service/src/lib.rs` keeps the public DTO and service envelope surface while becoming a small crate-root entrypoint.
- Protocol drift verification now scans split service implementation files for `handle_result` dispatch arms.
- `SUPPORTED_METHODS` remains in `crates/service/src/protocol.rs`; no protocol method names, payloads, or version values changed.

## Non-goals

- No service protocol method or payload changes.
- No protocol version bump.
- No provider default calls.
- No write/apply path.
- No hidden task state.
- No scanner/catalog fact mutation.
- No script execution.
- No credential reads.
- No raw prompt/response/trace persistence.
- No cloud sync.
- No telemetry.
- No public distribution, signing, notarization, DMG, or ZIP work.
