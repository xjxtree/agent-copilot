**Closeout 条件**

- `llm.previewPrompt`（或等效 preview 方法）与 `llm.confirmPromptAndSend`（或等效 confirmed send 方法）在 service protocol 中可用。
- 每次 provider 请求都能显示：
  - prompt scope 与 included/excluded 字段
  - redaction summary（隐私字段替换状态）
  - token/cost 估算
  - destination preview（provider/profile/base URL）
  - confirmation id（可复核）
- 发送请求前需要用户确认；确认后才允许发网络请求。
- 已确认调用有最小 redacted metadata 落盘（时间戳、provider、model、destination、token/cost、确认/Redaction 状态、状态/耗时/错误码），不存 raw prompt/response。
- 所有 provider-backed 返回都不触发 agent-config/snapshot/skill-content 写入，不触发 AI write-back，不触发脚本执行，不带入 telemetry。
- 若后续修改破坏上述闭环，V2.42 状态应重新打开并记录 blocker。
